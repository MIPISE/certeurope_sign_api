Onetime Triggered Signing API

This API provides access to signature platform

Workflow:

Depending on the needs of the client, the order and use of the calls to the API may differ.
Here is an example of a very basic workflow:

  1. Create Request - Provides the information that will be used for the signer’s certificate.
  2. Create Signature Request - Provides the information for the creation of the visible signature. Also provides the document(s) to be signed.
  3. Validate Signature Request - Uses provided information to sent one time password (OTP) by SMS or email;
  4. Launch Signature Request - Launches the signature process based on the two previous requests and received OTP.
  5. Get Signature Request Status - Returns the current status of the signature request and, if the status is “SIGNED”, provides the signed document. A list of all requests is provided below, however the order is not necessarily correct for the signature process.

We suggest that the client begins with this basic workflow and expands based on their specific needs.

API Endpoint ¶

The API PROD endpoint is https://sign.certeurope.fr/
The API PREPROD endpoint is https://sign-sandbox.certeurope.fr/

Authentication ¶

Certificate Authentication is used via a p12 type certificate that will be provided by CertEurope.

You may simply send this certificate with your requests or integrate it into a truststore should your application use one.
